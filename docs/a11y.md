# Accessibility Documentation

## Overview

This document outlines accessibility features and compliance for Contracting Co website, ensuring WCAG 2.2 AA standards.

## WCAG 2.2 Compliance

### Level AA Compliance
- ✅ **Perceivable**: All content accessible, alt text for images
- ✅ **Operable**: Keyboard navigation, focus indicators
- ✅ **Understandable**: Clear language, consistent navigation
- ✅ **Robust**: Semantic HTML, proper ARIA labels

## Keyboard Navigation

### Focus Order
- Logical tab order throughout the site
- Skip links for main content
- Focus visible on all interactive elements

### Keyboard Shortcuts
- `Tab`: Navigate forward
- `Shift+Tab`: Navigate backward
- `Enter/Space`: Activate buttons/links
- `Esc`: Close modals/dropdowns

### Focus Indicators
- 2px accent color ring
- 2px offset for visibility
- Applied to all interactive elements

## ARIA Labels & Semantics

### Navigation
- `aria-label` on menu toggle
- `aria-expanded` for dropdowns
- Proper heading hierarchy

### Forms
- Label association with inputs
- Error messages with `aria-describedby`
- Required field indicators
- Success/error state announcements

### Interactive Elements
- Buttons: Descriptive text
- Links: Contextual link text
- Images: Descriptive alt text

## Color Contrast

### Standards
- **Normal Text**: Minimum 4.5:1 contrast ratio
- **Large Text (18px+)**: Minimum 3:1 contrast ratio
- **Interactive Elements**: 4.5:1 minimum

### Color Combinations
- ✅ White text on dark backgrounds (ink, primary)
- ✅ Accent green on dark (22C55E / 0B1020) - 4.8:1
- ✅ Gold on dark (F59E0B / 0B1020) - 5.2:1

## Reduced Motion

### Implementation
- CSS media query: `@media (prefers-reduced-motion: reduce)`
- Animations disabled or minimized
- Transform animations reduced
- Page transitions simplified

### Components Affected
- Framer Motion animations
- Hover effects
- Scroll animations
- Count-up animations

## Screen Readers

### Semantic HTML
- Proper heading hierarchy (h1 → h2 → h3)
- Landmark regions (`<nav>`, `<main>`, `<footer>`)
- Lists for navigation and content groups
- Form fields properly labeled

### ARIA Enhancements
- `aria-live` regions for dynamic content
- `aria-hidden` for decorative elements
- `role` attributes where needed

## Forms

### Best Practices
- All inputs have associated labels
- Error messages clearly stated
- Validation feedback announced
- Required fields marked with asterisk and `aria-required`
- Field grouping with `fieldset` where appropriate

### RFQ Wizard
- Step indicators with progress
- Clear navigation between steps
- Form validation with clear errors
- Success confirmation

## Images

### Alt Text Guidelines
- **Decorative**: Empty alt (`alt=""`)
- **Informative**: Descriptive alt text
- **Complex**: Extended descriptions where needed
- **Logos**: Company/brand name

### Image Optimization
- Next.js Image component with automatic optimization
- Responsive images with proper sizes
- Lazy loading for performance
